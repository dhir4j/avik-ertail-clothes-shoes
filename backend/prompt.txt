# 7. UPI Payment Flow

Design a payment system using UPI.

Flow:

1. User places order
2. System creates order with `payment_status = pending`
3. User sees payment instructions with UPI ID
4. User pays via UPI app
5. User submits:

   * UPI reference ID
   * optional payment screenshot
6. Admin verifies payment
7. Admin updates order `payment_status = paid`
8. Order status changes to processing

Define database fields needed for UPI reference tracking.
